# 🚀 Crypto Lakehouse - Team Task Assignments

With **Phase 1** (Infrastructure), **Phase 2** (Ingestion), and **Phase 3** (Spark 3.5.8 & GCS Data Lake integration) successfully completed and validated, the core engineering foundation is solid! 

The remaining work to bring the Data Lakehouse to a production analytical state has been divided among three team members.

---

### 🧑‍💻 Teammate 1: Data Orchestrator (Focus: PySpark & Airflow)
**Assigned: Phase 4 & Phase 5**
Takes the cleansed Silver tables and transforms them into high-value aggregates, while automating the system's runtime.

* **Task 1: Gold Layer Aggregation (`silver_to_gold.py`)** 
  * Read from `gs://crypto-lakehouse-group8/silver`
  * Aggregate the ticks into 1-minute and 5-minute OHLCV (Open, High, Low, Close, Volume) candles.
  * Optionally calculate Moving Averages.
  * Save the results in Delta format to `gs://crypto-lakehouse-group8/gold`.
* **Task 2: Airflow Orchestration & Maintenance** 
  * Deploy Apache Airflow within Docker.
  * Write DAGs to schedule the `silver_to_gold.py` batch job.
  * Write a maintenance DAG to natively run `OPTIMIZE` and `VACUUM` commands on the Delta Lake to compress files and save GCS storage costs.

---

### 🛡️ Teammate 2: Platform Reliability Engineer (Focus: Cloud Migration & Metrics)
**Assigned: Core Migration Bridge & Phase 8**
Fixes the remaining legacy MinIO components and ensures the ecosystem runs reliably without crashing.

* **Task 1: HMS & Trino GCS Migration (Critical Path)**
  * Currently, `docker-compose.yml`, `hive-site.xml`, and Trino's `delta.properties` are hardcoded to the old local MinIO instance.
  * **Objective:** Update these files to connect securely to Google Cloud Storage using the Application Default Credentials (ADC), mapping them natively to `gs://`.
* **Task 2: Trino Catalog Sync**
  * Once connected, execute the Trino `CREATE TABLE` mapped queries to formally register the `bronze` and `silver` GCS directories into the PostgreSQL Hive Metastore so downstream tools can see the Delta tables.
* **Task 3: Monitoring Stack (Grafana / Prometheus)**
  * Configure Dashboards tracking **Kafka Consumer Lag** (so we know if Spark falls behind Binance WebSocket).
  * Configure Spark JVM Memory tracking (JMX) to prevent out-of-memory crashes.

---

### 📊 Teammate 3: Analytics Engineer (Focus: dbt & Power BI)
**Assigned: Phase 6 & Phase 7**
Serves the data to stakeholders and guarantees its absolute accuracy at the highest layer.

* **Task 1: dbt Data Quality Testing** 
  * Connect dbt Core to the Trino engine.
  * Write strict data quality tests on the Gold layer (e.g., ensuring `High` is never lower than `Low`, validating schema conformity, checking for null identifiers/timestamps).
* **Task 2: Power BI "Live Ticker" Dashboards**
  * Hook up Power BI (via DirectQuery or ODBC) to the Trino distributed SQL engine.
  * Build the final visualizations reporting off the `delta.default.gold_ohlcv` tables generated by Teammate 1.
